# Host UI

## Запуск

1. Установите зависимости и запустите сервер:

```bash
make run
```

2. Откройте Host UI в браузере:

```
http://127.0.0.1:8000/host
```

Если сервер работает на другом порту, замените `8000` на актуальное значение.

## Паринг по PIN

1. В Host UI в блоке «Авторизация» задайте PIN и нажмите «Создать токен».
2. Полученный host token сохранится в localStorage и будет подставляться для запросов.
3. Передайте PIN игрокам для подключения через их UI.

## Основные сценарии

- **Старт кампании**: запустить сервер, создать PIN для пары, раздать PIN игрокам.
- **Управление контентом**: работать с разделами Host UI (контакты, чат, игровые данные).
- **Работа с событиями**: следить за логом событий и реакциями клиентов в реальном времени.

## FAQ

**PIN нужно менять каждый раз?**
Нет, PIN можно использовать повторно, но по желанию его можно заменить, чтобы выдать новые токены.

**Где хранится host token?**
Токен сохраняется в localStorage браузера и автоматически применяется в запросах Host UI.

**Host UI можно открыть с другого устройства?**
Да, если сервер доступен по сети и разрешён доступ к `http://HOST_IP:PORT/host`.
